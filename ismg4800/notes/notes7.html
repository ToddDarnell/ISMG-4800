Dynamic HTML Object Model

  • The Dynamic HTML object model provides a way to control every HTML element in a Web page, including its attributes and CSS properties. 
    • Can create, move, and modify tags, attributes, images, objects, and text when and as needed. 
  • It is language independent allowing Web Developers can access and manipulate elements of a Web page using:
    • JavaScript, VisualBasic, Java, C++, CGI, Perl ...
  • The Dynamic Object Model is based on a W3C (WWW Consortium) specification.
    • Both Netscape and Microsoft support the Dynamic HTML Object Model and event generation.
    • Both browsers support load-time modifications, such as specifying fonts and other formatting information.
    • Only Microsoft supports runtime modifications that let DHTML perform multimedia effects, change the fonts on-the-fly, and move images.

Browser Detection

  • In order to run most Dynamic HTML scripts it is usually necessary to determine what browser and version is being used because code needs to be written differently depending on the browser being used.
  • Different browsers and different browsers implement the DOM differently.

navigator Object

  • The navigator object contains information about the Web browser that is viewing the page.
    • This includes the operating  browser name (appname), the userAgent and version number!
  • In the example below the navigator object is used to set a variable that can be used later to define the browser used.
    • If you run the code you will see the version number usually corresponds to the version of "mozilla" found in navigator.userAgent not the version of the browser software. 
    • You need to dissect the user agent string to find the actual browser version run.

<SCRIPT LANGUAGE = "JavaScript">
var browser = "none";
function browserDetect() 
{
  application    = navigator.appName;
  agent          = navigator.userAgent.toLowerCase();
  majorVersion   = parseInt(navigator.appVersion);
  minorVersion   = parseFloat(navigator.appVersion);  
 
  if ((agent.indexOf('mozilla') > -1) && 
     ((agent.indexOf('spoofer') < 0)
      && (agent.indexOf('compatible') < 0))) 
  {   
      if (majorVersion == 4) browser = "ns4";
      else if (majorVersion >= 5) browser = "ns6";
  }
  
  if (agent.indexOf("msie") != -1)
  { 
    if (majorVersion < 4) browser="ie3";
    else if (majorVersion == 4) 
    { if (agent.indexOf("msie 6.0")!=-1) browser="ie6"
      else if (agent.indexOf("msie 5.0")< 0) browser="ie4";
      else if (agent.indexOf("msie 5.0")>-1) browser="ie5";
      else if (agent.indexOf("msie 5.5")>-1) browser="ie55";
    }
  }
}
</SCRIPT>

DOM Detection

  • An alternative to detecting the browser using the navigator object, is to detect which DOM is being used.  There are 3 principle different DOM in use:
    • Internet Explorer DOM
    • Netscape 4 DOM
    • W3C Dom (used in Netscape 6)
  • To detect what DOM is being used you can create boolean variables based on whether a certain "non-standard" DOM feature is found:

var NS4DOM = document.layers ?true : false;
var IEDOM  = document.all ?true : false;
var W3CDOM = ((document.getElementById)&&(!IEDOM))?true:false;


Element Interfaces

  • Every pair of tags on a Web page bound a "element" that can be referenced via the Dynamic HTML Object Model.
  • Every element is an object with:
    • Attributes, style, content and events
  • Some elements offer extra members:
    • <FORM> exposes an elements collection (everything on the form), submit and reset methods
    • <TABLE> exposes a collection of rows.  Each row exposes a collection of cells.
    • <INPUT>, <TEXTAREA>, <SELECT> etc. extra control over the attributes and contents.
  • In order to "do something" with an object found on a HTML Web page, you first need to be able to reference it (i.e. tell the script which element you are talking about).

By ID

  • The simplest way to reference an element is by its ID attribute.  The 3 different DOMs support referencing Objects by ID differently:
    • NS4DOM: document.id
    • IEDOM: id
    • W3CDOM: document.getElementByID(id)
  • This script calls a method "getObject()" to retrieve a given page element by ID and then  dynamically changed the contents of <div> "pText" from "Welcome to our Web page to!" to "Thanks for coming." using the innerHTML property of the paragraph (innerHTML refers to the text contained in that element).

CSS used with Dynamic Positioning

  • Cascading Style Sheets is also used in Dynamic HTML to control the position, layer an visibility of objects.
  • Box properties allow the properties of HTML box elements to be set.  These elements include <DIV>, <TABLE>, <IMG> etc.
    • The primary Box Properties are width and height which allow you to set the dimensions of the box element in either pixels or as a percent of the available space.
    • overflow: When an element does not fit in the defined width or height overflow determines whether the remaining part of the element should be auto,hidden, scroll or visible. 

  • Cascading Style Sheets also allow more control over the position of elements on an HTML page using the position property. 
    • The default position property is "static", which places an element wherever you wrote the code. 
    • "absolute"  or "relative" positioning allows a section to be placed exactly where you want it using the number of pixels or percent from the top & left of the window or relative to the current location. 
    • "z-index" allows elements to overlap with higher z-index elements on top of lower ones.
    • Position properties work with both images and <div> elements in Internet Explorer.  However, they only work with <div> elements in Netscape.

Dynamic Positioning

  • You can also use JavaScript to dynamically position elements on a Web page. However, all 3 DOMs handle the dynamic positioning of objects differently.
  • The shiftIt() method below contains code to shift an object in X & Y -
    • It is the same code as found in the BrowserAPI.js discussed in your book. 

function shiftIt(id, dx, dy)
{
  var object=getObject(id);
  if(NS4DOM) object.moveBy(dx,dy);
  else if(IEDOM)
  { object.style.left =object.style.pixelLeft+dx;
    object.style.top =object.style.pixelTop+dy;
  }
  else if(W3CDOM)
  { object.style.left=parseInt(object.style.left)+dx;
    object.style.top=parseInt(object.style.top)+dy;
  }
}

Example

  • For example: The functions below are used to update the position of a <div> element with id=tMove (the Welcome! at the top of the page).
    • Function startMove('tmove'); is called in the onload event of the <body> tag.  It calls function setInterval which takes a function name and how often to run that function (100 ms).
    • Function moveText takes a HTML element and moves it across the window. The distance it travels across the screen is based on the page_width (calculated differently for different browsers) minus a certain number of pixels.
    • Once count reaches a multiple of the traverselength, the text reverses directions and changes color.
    • moveText calls shiftIt to actually move the <div> element
var item_width = 500;
var margin = 35;
var speed = 10;
var count = 0;
var direction = 1;

// find screen size
var page_width=screen.width;
if(IEDOM) page_width=document.body.clientWidth;
else if(W3CDOM) page_width=innerWidth;
traverselenght = page_width - item_width/2-margin;

function startMove(id)
{  window.setInterval( "moveObject('" +id+ "')", 100 ); }

function moveObject(id)
{
   count += speed;
   if ( ( count % (traverselenght) ) <
         Math.abs(speed) && count % (traverselenght) >= 0 )
   {
     speed *= -1;
     direction = !direction;
     var col = ( speed < 0 ) ? "red" : "blue"
     var object=getObject(id);
     if(NS4DOM) object.color = col;
     else if(IEDOM || W3CDOM) object.style.color = col;
   }
   shiftIt(id, speed, 0);
}

var menuwidth=150;
var rightboundary=menuwidth*.9;
var leftboundary=-1.1*menuwidth
var themenu;
var menuname;

// set the menu allows you to work with multiple menu names
function setthemenu(id)
{ menuname=id;
  return getObject(id);
}

// begin to pull menu out by calling drawengine every 10ms
function pull(id){
  // if "pulling out" a new menu immediately hide old one
  if(menuname != id) placeObject(leftboundary, yCoord(id)) ;
  if(id != null) setthemenu(id);
  if (window.drawit)
    clearInterval(drawit)
  pullit=setInterval("pullengine('" + id + "')",10)
}

// begin to draw menu back by calling drawengine every 1ms
function draw(id){
  if(id != null) setthemenu(id);
  clearInterval(pullit)
  drawit=setInterval("drawengine('" + id + "')",1)
}

// pull menu out 5 pixels at a time
function pullengine(id){
  if (xCoord(id)<rightboundary)
    shiftIt(menuname,5,0);
}

// draw menu back 5 pixels at a time
function drawengine(id){
  if (xCoord(id)>leftboundary)
    shiftIt(menuname,-5,0);
}

Other  Events

  • Both fourth generation browsers introduced a new player onto the event handling stage: the event object. 
  • The event object, then, contains information about the event. Which event? The event which has just happened!
  • Put another way, the event object is a snapshot of details surrounding an event immediately after it has happened. 
  • You can use the event object in the event handler code to gain insight into details of the event such as where the mouse pointer was when the event took place, whether certain keyboard keys such as Shift and Ctrl were depressed at the time, and so on.

Event Object Netscape vs. Microsoft

  • Microsoft and Netscape support a very different set of events and available properties for their respective event objects.  
  • A complete list of these two different event sets can and their respective properties can be found on the attached event list. 
  • Netscape and Microsoft also support a completely different method of handling their events:
  • Netscape Navigator supports "Event Capturing", or the ability to intercept events before they reach their target and potentially re-route or even cancel them. To begin, we must enable event capturing at one or more of three possible "levels": the window, the document, or the layer.
  • Microsoft Internet Explorer supports "Event Bubbling." In this paradigm and event is similar to an air bubble trapped inside a body of water -- it floats towards the top of the object hierarchy.
  • The message:  If you are going to use the more advanced event features you will need to write completely different code for Netscape and Microsoft or be operating in a single browser domain!
