Web Server Operating Systems

  • There's so much emphasis placed on specific applications that we tend to forget the engines that drive them: operating systems and platforms. 
  • When evaluating performance, you need to look at both the Web Server application and the operating system. 
    • A fast Web server on a slow OS will still be slow.
  • Some basic operating systems Web Servers are used with are:
    • Commercial Unix (Sun Solaris)
    • Open Source Unix (Open Linux, Open BSD)
    • Windows (NT and 2000)
    • Macintosh (Mac OS X)
    • Novell NetWare
  • All of these operating systems got pretty good ratings from Web Server Compare

Choosing an Internet Service Provider

  • If your company or organization is not large enough or does not want to maintain their own Web server then you need to select an Internet Service Provider (ISP) to host your Web site.
  • Even if you are maintaining your own Web Server you still will need to get it connected to the Internet (so some of the ISP issues are relevant).
  • The following are some of the features you should look for when shopping for an ISP:

Easy Connection

  • Choose an ISP that allows you to easily connect to their service.  
  • For smaller sites you will want to be able to connect using a local phone call.  
    • The provider should have enough Points of Presence (POPs) to make dialing easy.
    • You should never get a busy signal when you dial your ISP - if you do change ISPs.
    • Try to match the size of the ISP to the size of the company.  A local company does not need a national ISP.
  • For larger sites you may need a faster connection to the internet or one that supports more users.  You have a few options - each depends on local availablitiy:
    • ISDN stands for Integrated Services Digital Network. 
      • The top speed of an ISDN line is 128k/s in both directions. 
      • ISDN is a dial-up service like traditional modems. 
    • Cable modems are devices that gives internet connectivity through a cable television connection. 
      • A cable modem can provide connection speeds of anywhere from 500K/second to 20+Mb/second. 
      • A cable modem provides "always on" service.  
      • A single cable modem can also be shared (via a network server) among multiple computers, providing simultaneous Internet access for a small network.
      • Cable is a shared network, when too many users on one line try to share the same cable, performance can suffer dramatically. 
      • Cable customers are forbidden to telecommute, host web sites, use video teleconferencing and many other bandwidth intensive applications. 
      • There are also certain security risks which can make users more vulnerable to data interception, unauthorized monitoring and hacking from other users along the same cable network.
    • DSL, or Digital Subscriber Line, is another option for faster internet connections. 
      • Commonly, the affordable DSL solutions for go up to 2-3 Mb/second at the most.
      • DSL provides "always on" service.
      • DSL provides a dedicated line to each subscriber, making it more appropriate for businesses than cable modems.
      • There are rarely restrictions on DSL customers using bandwidth intensive applications.

Accessible Technical Support

  • Technical support is an absolute necessity! Make sure your ISP has competent, accessible customer service.
  • Call potential ISPs and talk with someone in customer service.  
    • Tell them what you hope to accomplish (such as set-up a Web site, run ASP etc.) and how experienced you are and see how helpful they are.
    • Time how long they keep you on hold when you are waiting to talk to customer service.
    • Local ISPs may not have a large staff but they also have fewer subscribers - National ISPs have so much volume they may keep you waiting an unacceptable length of time.

Additional E-mail Addresses

  • All access accounts come with at least one email address.  If you are part of a group you may want additional addresses for your organization.

 Personal vs. Commercial Accounts

  • Personal ISP accounts generally are less expensive than a business account.  You have less disk space, fewer features and a more complex URL.  
    • Businesses generally will want more services than those provided with personal ISP accounts when establishing a commercial Web site.
  • Commercial ISP accounts usually provide extra services including:
    • SQL Database Support: There are several products (e.g. ASP or Cold Fusion) that allow databases to be connected to the Web.  A commercial ISP account should support/allow some type of SQL Database connectivity.
    • Secure Socket Layer (SSL) Support:  SSL is an Internet communications protocol that allows encrypted transmission of data between the user and the server.  SSL is necessary if you are going to take orders via the Web or transmit other types of sensitive data.
    • Shopping Cart Hosting: Some ISPs/external companies will also host your shopping cart. This includes taking secure orders, maintaining credit card information in a secure database, submitting payments to the  credit card companies etc.
    • Multimedia Capabilities: Some ISPs will support the streaming of multimedia.  Streaming is a server based technology that transmits audio or video in a continuous stream  so it can be played while it is still downloading. 
    • CGI Script Capabilities: You will want your ISP to allow you to use CGI scripts on you Web site.  You will need to find out if you will be allowed to create your own CGIs, whether your CGIs can be put in your own Web spece or if they will need to go in the cgi-bin directory, or if the ISP has generic CGIs that you can use (e.g. to mail forms).
    • Activity Reporting: Most Web servers can track how often users visit your site, which pages are requested most often, and how your Web site traffic varies from month to month. 

Buying A Domain Name

  • When establishing a commercial Web site you will usually want to get a domain name for your site (mycompany.com).
  • If you want to  see if a domain name is available , or to register your own domain name, visit Network Solutions.
  • For an additional fee your ISP can register your domain name for you.  Registration requires a valid name and the servers primary and secondary IP addresses.
  • Domain names need to be renewed every two to five years.

Web Site Testing

Why Test?

  • Whenever someone visits your Web site, they judge your company.
  • A site that is fast, informative and easy to use adds real value to your brand. What's more, it will generate repeat visits, new business, valuable referrals and positive PR.
  • But you don't know who the visitors will be. They won't always be using the most popular or most recent computer systems, browsers or plug-ins, for example. Some combinations may cause problems. Your visitors may be experienced Web surfers, or they may not. Some will connect through high-speed networks, perhaps with firewalls and content filters, and others will have slow dial up connections. 8% of your male visitors will be color blind.
  • Your aim should be to cater to everyone. So how can you make sure your site really works?
  • You can't rely on your customers to take the time to report any hidden problems with your site - they'll simply click and go.
  • You need a more scientific approach.

User Experience Testing

  • People have short attention spans and the Web gives them plenty of choice.
  • You should test that your site can be used by your untrained customers without assistance.
  • Ensure that functionality has appropriate depth, pages operate as expected and help is available when required.

User Testing

  • Test with a variety of users from your target audience to get their feedback on your Web site.  Get subjects with:
    • A variety of backgrounds
    • Different computer skill levels
    • Different familiarity with the material / products
    • Avoid friends as testers because they may not be objective
  • If possible, watch users interact with your site to see where they might have difficulty navigating / using your site.
  • Formalize your testing by creating replicable methods of testing your site.  This lets you compare results across users and assess improvements as you refine the site.
    • Prepare a series of questions to ask users after viewing the site
    • Give them a specific task to complete or have them find a specific piece of information then have them rate the ease or complexity of completing the task.  You can also time how long it takes them to complete the task.
  • Develop a Feedback form that users can fill out after testing the site (it can be online too).  Include a set of criteria and let them rate the site on a progressive scale (1-5).  Questions could include:
    • Did you find the information you needed?
    • Was it easy or difficult to access the information you needed?
    • Did you find the Web site visually attractive?
    • Did you find the content easy to read?
    • Did you find the Web site easy to navigate?
    • Did you think the information was presented correctly?
    • Did the information have enough depth?
    • What area of the Web site did you like best? Why?
    • What area of the Web site did you like least? Why?
    • Would you recommend the Web site to others?

Content Testing

  • Maintain the quality of your content - not only do links go bad, other information can become obsolete or incorrect.
  • Periodically check you content to make sure it still makes sense.  
    • Are your prices competitive?
    • Are you referring to old versions or obsolete products?
    • Is your corporate information up to date?
    • Are you using an old/obsolete technology to provide the information?
    • Are your images or color scheme dated?
  • Poorly maintained content can degrade your organizations image.  Costing you visitors, customers, and investors. 

Usability Testing

  • Usability testing is done to ensure you do not lose customers because they find your site hard to use.
  • Usability can be thought of as a measure of how simple it is for the visitor to complete the sequence of steps necessary to accomplish any given task. The smaller the number of steps and the more intuitive each step is, the more usable that function of the web site is.
  • Usability of a Web Site is determined by user satisfaction and ease of navigation. A user shouldn't have to remember the sites organization or functionalities. The goal is user effectiveness and efficiency with a minimal likelihood of errors while performing the tasks the site has been designed for; finding information needed or completing an e-commerce transaction.
  • Usability is very much like quality: you perceive it mainly when it is missing.
  • Categorically, sites score poorly in:
    • consistency of presentation and controls across the site
    • natural organization of information: clear structure, systematic identification, clear and meaningful labels
      contextual navigation, in terms of how much information is given for providing a context to the user: where is s/he? where can s/he go? how can s/he go back?
    • efficient navigation, in terms of time and effort the user needs to move around the site
      adequacy of feedback in displaying the effects of some command, in answering to requests
      search ability, or how effectively the site content can be sought in search engines

Accessibility

  • Making your web pages accessible is not only a good idea, but it's good for business.
  • Access Now, Inc. and Robert Gumson have sued Southwest and American airlines because Southwest's site is allegedly "inaccessible". This isn't the first time something like this happened, either. In 1999, the National Federation of the Blind filed a lawsuit against AOL because AOL's internet services were inaccessible. Over the past two years, Access Now has also sued Barnes & Noble and claire's because their sites were inaccessible.
  • The first thing to do to improve Web site accessibility is to use the ALT attribute for images that convey information. While ALT attributes aren't necessary for spacer images or purely ornamental images, ALT attributes are essential for all images that convey content. The text in an ALT attributes describes to users of non-graphical browsers (including screen readers) what information is displayed by the image.
  • For accessibility purposes, people who create forms may need to add extra markup to help screen readers translate a visually-based association of a text label (like a question or description) and its form input field into a literal association of these elements.
    • For example:Enclose the question or description associated with a form field in a <label for=""> tag. Then, add an ID attribute to the input field and make both the ID and FOR equal to the same value.

    <label for="firstName">First Name</label>
    <input type="text" name="fName" id="firstName" size="10" maxlength="20" value="foo">

    • If the form elements are laid out correctly on a page by using tables or CSS, adding that extra markup isn't as important. What's important is that the users know what the question or description is for a form's field immediately before they're prompted to enter a response.
  • Obeying other basic Usability rules also helps improve Web site Accessibility

Link / Image Testing

  • Make sure all links (including those to external cascading style sheet or JavaScript files) are fully functional.  
  • Test your links immediately after you upload and regularly thereafter (monthly?)
  • There are some Web robots, spiders, snakes or crawlers that will search you Web site to make sure all your links are valid (they do not check to see if the content has changed).  

Compatibility Testing

  • With all of the different browsers and operating systems on the market today, it is crucial to verify that your web site or web application functions properly across a wide range of platforms.
  • By testing your web application on different browsers and operating systems, you can ensure that certain coding such as Java Applets, Active X Controls, JavaScript, CSS, and HTML function properly across multiple configurations.
  • Test in as many different environments as possible, including:
  • Multiple browsers and versions:  Don't just test the current version of Internet Explorer and Navigator, test older versions too.
  • Multiple Operating Systems: Test on PCs, Macintosh, and Unix Systems.
  • Connection Speeds: Test for a variety of connection speeds. Got to a friends house, library or Internet cafe and test from there.  According to Wired news, 33% of Web users will leave a Web site if it takes longer than 8 seconds to load.
  • Display Types: Test using different screen sizes, screen resolutions and color depths to make sure your site/colors display well at each setting.
  • Plugins: You need to test any Plugins used in all environments.  Including:
    • Checking the use of various browser plug-ins
    • Checking the absence of various browser plug-ins

Optimizing Your Web Pages

  • If your web pages are optimized, they'll be easier to maintain and, most importantly, they'll load faster. Remember, nobody ever complained that a site loaded too quickly.

Graphics should be optimized to save bandwidth and reduce download time.

  • Just because Jesus miraculously turned water into wine doesn't mean he can miraculously turn your 1280- x 1024-pixel image whose file size is 1.8Mb into an image whose file size is only 74Kb just because you changed the WIDTH= and HEIGHT= attributes to WIDTH="420" and HEIGHT="336".
  • Easy Thumbnails Can be used to reduce the size of the picture to the size specified in the IMG tag's attributes. You can reduced the 1.8Mb picture down to 74Kb just by making a "thumbnail."
  • IrfanView IrfanView is an extremely valuable graphics tool. It allows you to size the imiga nad save it as a "small" file - further reducing your image size.
  • Dr. HTML offers a free (five-page per day) analysis of your web page. Once you find the mistakes, it's up to you to fix them.
  • Web Page Analyzer allows you to enter a URL to calculate page size, composition, and download time. The script calculates the size of individual elements and finds the total for each type of web page component. Based on these page characteristics the script then offers advice on how to improve page display time. The script incorporates best practices from HCI research into its recommendations."
  • Screen Calipers According to the site: "The Screen Calipers are a unique on-screen measurement tool, similar to a screen ruler. They resemble real life calipers in both form and function." They are useful when you want to see how a page looks in certain size windows.

Search Engine Placement

  • If Google can't find you, then you don't exist.

  • You can lose customers before they even get to your site.
  • It is now widely accepted that if you are not listed in the first 3 pages of results from a search engine then you will not be found by that potential customer.
  • The majority of people will simply go to a competitor's site that does appear in those results or try new search criteria assuming that they have failed to find what they are looking for.
  • Before you register your Web site with a Search Engine you should enhance your Web site to make sure it displays well within Search Engine results.
  • Use Meaningful Titles: All pages on your Web site need meaningful titles.  Some search engines read only the title when indexing the Web site.  The title is also what shows up in a user's favorites or bookmark list.
  • Use <META> elements:  You can use META elements to raise on your Web site listing with certain search engines.  AltaVista, Excite, Inktomi and HotBot use them but some other search engines ignore them. 
    • <META> Elements go in the <HEAD> section of the document and let you specify information about the document that is invisible to the user.
    • Common <META> elements are Author, Description, Keywords and Generator.
    • Sample <META> elements are shown below:
<HTML>
<HEAD>
  <META name="Author" content="Dawn Gregg">
  <META name="Description" content="Guidelines for 
              publishing and maintaining your Web site">
  <META name="Keywords" content="WWW, ISP, Web Server, 
              Testing, Search Engines, Web Site Design">
  <META name="Generator" content="Microsoft FrontPage 4.0">
</HEAD>
  • Be careful with Frames: Avoid using a Frameset page as the top page on your Web site because frameset pages contain no content for a search engine to search.
    • If you need to use frames on an page you want indexed use <META> elements to describe the site, have a meaningful title, add additional information in the <NOFRAMES> part of the page:
</HEAD>
<frameset cols="17%,83%" BORDER=0 frameborder="0">
<frame SRC="menu.html" NAME="menu" NORESIZE BORDER=0>
<frame SRC="main.html" NAME="main" NORESIZE BORDER=0>
<NOFRAMES>
  <BODY>
     <P>ISMG 6240: An introduction to Web authoring
        and electronic commerce.  Course Web site
        at Univeristy of Colorado Denver</p>
     <a href=main.html>No Frames Version of the Site</a>
  </BODY>
</FRAMESET>
</HTML>   
  • Use ALT text with images: Always add alt text to all of the graphics on your page.  Some search engines will read the contents of the ALT attribute, which is especially useful if you start your page with a graphic

<img alt="CGI" src="CGIlogo.gif">

  • Once you have completed all the steps to ensure that your Web site will display well with Search Engines you can register your site with each popular search engine.
    • Most have a link on their main page that allows you to register your site.
    • Allow some time to pass then go to each search engine and check to see if your Web site has been indexed.  Use pertinent search terms for your site.
    • You may need to resubmit your URL if your site does not come up.  
  • You should conduct periodic reviews of your Web site's current positions with specified search engines and the keywords with which you would like, or expect, to be identified, together with the positions of their main competitors for the same or similar keyword/engine combinations.

Search Engine Optimization

  • The Keyword Verification tool  checks to see if your site is in the top three pages of a search engine result for a certain keyword. The reason it only check the top 3 pages is because most queries don’t go past the 3rd page of search results.
  • The Search Engine Saturation tool  refers to the number of pages a given search engine has in its index for your website domain. Check your Search Engine Saturation and compare your website to competitors. Check this information on a regular basis to see if your Search Engine Saturation is climbing or falling.
  •  The Link Popularity tool helps search engine optimizers and online marketers to find out who is linking to your site, but to also run a benchmarking report to show you quickly where you stand in comparison to competitors and other major online players.
