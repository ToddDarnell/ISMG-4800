Cascading Style Sheets

What are Style Sheets?

  • Cascading Style Sheets separate the appearance of web pages from the content of web pages. 
    • This means that the information goes into HTML files, but HTML files should not contain information about how that information is displayed. 
    • Information about how the pages should appear goes into style sheet files.

Advantages of Style Sheets

  • The traditional HTML approach is to "hardcode" all of the appearance information about a page. 
    • eg. Every heading, in every page, the font size and face properties etc. need to be set. 
    • As the page count increases the editing/re-editing times increases as does the possibility for introducing errors.
  • Cascading Style Sheets allow formats for headings etc. to be set once. 
  • They allow whole organizations to share style sheets, ensuring consistency across the site.

Using the Style Attribute (Inline Styles)

  • Styles can be declared inside an HTML tag by adding the style=" " attribute:  
           style="property:value" 
    • eg. for red text in a paragraph:  
      <p style="color:red">Totally red!</p>       

      Totally red!

    • To add more than one property in a style. Put a ";" after your first property:value, & add another: 
      <p style="color:red; font-style:italic"> Red-hot italic!</p> 

      Red-hot italic!

Embedded Styles

  • Styles can also be declared for an entire document.
  • These styles are declared inside the head section of the page and begin with the <STYLE> tag and end with the </STYLE> tag.
  • Below is an example that declares the style for
    the <P> tag in the head of the document: 
<HEAD> 
<STYLE type="text/css">Begin style definition
<!--                   Comment to hide style from older browsers
P { color:purple;      Set text color to purple for a paragraph
    font-style:italic; Add italics 
    font-size: 1.5em;  font 1.5*surrounding size 
  }                    End definition  for a span
// -->                 End comment to hide style
</STYLE>               End style definition
</HEAD> 
  • The following HTML in the body of the document:

<P>Big and italic paragraph</P> 

  • Yields:

    • Big and italic paragraph .............!

Adding Style to Multiple Tags

  • Styles can also be declared for multiple tags at once.
  • Below is an example that declares the style for
    the <h1>, <h2> & <h3> tags : 
<HEAD> 
<STYLE type="text/css">Begin style definition
<!--                   Comment to hide style from older browsers
h1, h2 { color:lime;   Set text color to lime for 2 headings
         font-weight:bold; Add bold 
       }                   End definition  for h1, h2 style
// -->                 End comment to hide style
</STYLE>               End style definition
</HEAD>

Using External Style Sheets

  • External style sheets allow the use of one group of styles on many pages at once. 
  • External style sheets are TEXT files ending with the .css extension. 
  • They just contain style definitions for different types of elements.  They DO NOT contain <style> & </style> tags.
  • The following was saved in a file style1.css and imported into in a file test1CSS.html.
P      { font-family: Arial, sans-serif;
         color: blue;
         font-size: 1.17em; } <!-- Approximately 14pt -->

EM     { color:red;
         font-size:1.17em;}

STRONG { color: green;
         background-color:#ccffcc;} 

Using <Link> to Use an External Style Sheet

  • To link to or import a style sheet, you need to add the <LINK> tag inside the head section of your html document: 
    <LINK rel="stylesheet" type="text/css" href="styles/style1.css"> 
  • Now, any time you use the <P> tag in one of your pages linking to the style sheet, you will get:
    • Blue, 1.17em (approximately 14 point), Arial text (or if Arial is not available you will get the default sans-serif font)
      <P>I am Arial!</P> 

I am Arial

  • When you use <strong>, you will get a green font, & green background:  
    • <p>This line is <strong>really</strong> important!</p> 
This line is really important!
  • When you use <em>, you will get a red font & 1.17em (1.17*neighboring size): 
    • <p>This line is <em>really</em> important!</p> 

This line is also important!

Using @import to Use an External Style Sheet

  • You can also include a stylesheet using the @import statement instead of using the special <link> tag (above).
  • To import a stylesheet into a HTML page do the following:
    <style>
       @import url(styles/style1.css);
    <style>
  • You can also import Eternal Style Sheets into .css files:
       @import url(styles/style1.css);
       other style declarations...
  • Advantage of using @import in other style sheets is it allows you to build a set of "generic" styles that you use always and then to add to those styles for a specific application.
    • e.g. You can create a stylesheet for all pages on a Web site and then create special style sheets for product pages, information pages, internal pages external pages ...  that imports the baseline styles.
  • Another use for the @import statement is to quickly allow you to change your site color scheme - perhaps to adapt to changing seasons...

Style Inheritance

  • Styles inherit attributes from the tags they are within.
  • If you set the font-color property to AQUA in the <body> tag - then all text on the page will be AQUA unless the font color is explicitly set for that object.
  • In the previous example - style1.css sets properties for <p>, <em> and <strong>.  The page test1CSS.html shows how <body> properties are inherited by the <p> and how <p> properties are inherited by the <em> & <strong> tags...

Font Properties

  • You have seen many of the basic font properties already, 
    • font-family: eg. Arial (sans-serif), Times (serif) and Courier (monospaced). 
      Include the generic font family name (serif etc.) so an alternate font can be used if the one specified is not available.  
    • font-weight: normal, bold, bolder, lighter, 100-900
    • font-style: normal, italic, or oblique
    • font-size: in inches (0.5in) points (14pt), pixels (40px) or percent (1.5em or 1.5ex).   
    • font-variant: normal or small-caps
H6       {font-family: Courier New, Courier, monospaced;
          font-style: oblique; 
          font-weight: bolder; 
          font-size: 2em;
          font-variant:small-caps; }
  • And when you use the <H6>tag you will test all the font properties: 
    • <H6>This is a test!!!<H6>
This is a test!!!
  • Note - it is best to specify font sizes in percent (em) because these font sizes will resize based on user preferences. Thus, if someone selects View -> Text Size -> largest in Internet Explorer - the text size will increase.  If you specify a fixed size in inches, points or pixels it will not.

Text Properties

    • letter-spacing, text-align, vertical-align, text-decoration, text-transform, text-indent, line-height

Using Classes to Define Styles

  • Style classes allow styles to be defined without using up a tag each time you wish to have a new style. 
  • Define 2 classes,  one gives you a red font, another gives you a green font, both w/ background color by:

<STYLE type="text/css"> <!-- 
  .redfont   { color:red;Note a "." immediately precedes the class name
               background-color:#ffcccc;}
  .greenfont { color: green;
               background-color:#ccffcc;}  
--></STYLE> 
</HEAD> 
  • If you would like to have a section with a red font and a section with a green font, but wish to use the <p> tag for both, you can!
    <p class="redfont">I am red, & I'm important</p> 
    <p class="greenfont">I am green, so there</p>
     Note: No "." when the class name is used!
  • This gives: 

    I am red, & I'm important

    I am green, so there

Box Properties

Forms

  • Web forms are used either to return information to a Web server for some action or to submit information to a local client function (JavaScript or VBScript).
  • The form starts with a <form> tag and ends with </form>. Here's what it looks like:

<FORM NAME="frmInput" METHOD="POST" ACTION="MAILTO:me@myserver.com"> 

  • The NAME attribute specifies the forms internal identifier within the Web page (It is similar to the NAME property in Visual Basic).
  • The ACTION attribute indicates the action taken when the form is completed.  In this case "MAILTO:me@myserver.com" denotes this is a mailable form and the data is sent to me@myserver.com.
  • Forms can have METHOD="POST" or METHOD="GET" attributes.  Some form processing programs are designed to work with post, some work with get & some work with both.  The method is interpreted by HTTP and controls how the data is returned to the server based program.
    • POST writes the data back to the server after an initial connection is established. Essentially the browser begins talking to the Web URL (specified in action) and then sends the data.
    • GET uses URL rewriting so the data is sent at the same time the connection to the page is established. With get you will see the form data displayed in the URL as parameters for the page that is returned:

    <FORM NAME="frmInput2" METHOD="Get" ACTION="http://carbon.cudenver.edu/~dgregg/ismg4800/readform.php">
      <input type="text" name=myname value="dawn">
      <input type=submit>
    </form>

Sends this URL:
http://carbon.cudenver.edu/~dgregg/ismg4800/readform.php?myname=dawn

Input Elements

  • The <INPUT> tag is used to create a new Input field, hidden field, password field, check box, radio button, command button, submit button or reset button.
    • The TYPE attribute of the <INPUT> tag determines the type of input control that is being created.
    • The NAME attribute specifies the fields internal identifier within the Web Page.
    • The VALUE attribute specifies the control's current or default value. 
  • <input type=text>: Allows a user to whatever data they want into a text field. 
    • The maximum number of characters that can be entered in these text fields can be set using the attribute maxlength
    • If an initial value is supplied for the text field it will appear in the form as the default value for the field.
<input type="text" name="name" size="12" maxlength="20" value="ismg 4800">
  • <input type=password>: is just like a text field - except that for each character typed an * appears in the text box instead of the actual letter.  It is good for accepting passwords or credit card numbers online.  However, they do not "encrypt" the text passed so by them selves, they are not secure.
  • <input type=hidden>: Hidden text fields do not accept user input.  Instead they are used to store data in a form but not display it in the browser.  Hidden form elements are frequently used on multi-page forms where the information entered on the first page of the form is stored in "hidden" form elements so it can be submitted to the server with all of the data on the second page of the form.

Form Labels

  • On most Web pages the label for each form element was simply written in HTML immediately prior to the form element.
  • For accessibility purposes, people who create forms may need to add extra markup to help screen readers translate a visually-based association of a text label (like a question or description) and its form input field into a literal association of these elements.
  • For example: Enclose the question or description associated with a form field in a <label for=""> tag. Then, add an ID attribute to the input field and make both the ID and FOR equal to the same value.

    <label for="name">Your Name</label>
    <input type="text" name="name" id="name" size="10">

  • Once a label is created - if you click on it Internet Explorer will place the curser in the appropriate text box.
  • If the form elements are laid out correctly on a page by using tables or CSS, adding that extra markup isn't as important. What's important is that the users know what the question or description is for a form's field immediately before they're prompted to enter a response.

Text Areas 

  • Text areas are multi-line form elements, frequently used for capturing comments or displaying output from client side programs.
  • The generic form for a TEXTAREA is:
    <textarea name=comment rows=5 cols=80></textarea>
    Where rows is the number of lines of text you want displayed and cols represents the number of characters per line.
  • If you want default text to appear in the text area - just enter the text between the opening <textarea> tag and the closing </textarea> tag.
  • You can also control whether or not the text area automatically wraps the text input by setting the wrap attribute to on (default), off, soft or hard.

Form Buttons 

  • There are 4 types of buttons that can appear on a form.
  • <input type=button> does nothing unless a client side script is written to cause it to do something when it is "clicked".
  • <input type=submit> is a special button that submits the form somewhere for processing.
    • The form can be mailed to an email address, sent to a CGI program, sent to a JavaServerPage, to ASP, to PHP, to Perl, to Python etc...
    • The form is sent to the location specified in the action attribute of the <form> tag.
      <form method=get action="mailto:dawn.gregg@cudenver.edu"> would send the form to me via email.
  • <input type=reset> just resets all of the values in the form to their default values.
  • <input type=file name=filename> creates a button that opens a file dialog box that allows a user to browse for a file name on the local computer:
  • If you specify a value=??? for any of the above button types - that value will be the label shown on the button.
  • You can also create "Custom" submit, reset and button buttons by using the
    <button> tag instead of the <input> tag:
    <button name=gobutton type=submit>
       <img src=../images/go-button.gif>
       Search This Site
    </button>

Image Buttons

    <input type=image src=../images/go-button.gif  name=go value=go>
     
  • The x & y location (in pixels) of where the button is clicked is also submitted with the form so they can be used to send image map type data back to the server.
