More Browser & JavaScript Objects

Browser Objects and Document Object Model

  • Access to browser objects, and their properties, methods and events is controlled (specified based on) the Document Object Model (DOM).
  • The DOM maps the Web page for developers.
    • It details the characteristic properties of each element of a Web page, allowing developers to manipulate HTML components and, in turn, manipulate the page.
  • Though the DOM for Internet Explorer and Netscape are similar - they also differ in many significant ways! Thus, some functions you write will work differently (or not at all) in different browsers and browser versions.

DOM Objects, Properties, Methods, Events

  • Objects: Container which reflects a particular element of a HTML page;
    • Objects "contain" the various characteristics which apply to that element (known as properties and methods);
    • Access to an object's properties and methods is by a "." operator. 
  • Properties: Characteristics of an object;
    • The document object possesses a bgColor property which reflects the background color of the page.
    • The document property bgColor can be accessed with
               document.bgColor = #FFFFFF;
    • Using a programming language (e.g. JavaScript) you may read or modify the color of the current page.
    • Some objects contain very many properties, some contain very few. Some properties are read-only while others can be modified, possibly resulting in immediate on-screen results.
  • Methods: A method is a function associated with the object. Sometimes the method also returns a result value.
    • Methods are triggered by the programming language being used, such as JavaScript.
    • Method calls have () after them which may or may not contain arguments. 
    • For example: The window object possesses a method named alert(). When supplied with string data, the alert() method causes a window to pop up on the screen containing the data as its message; e.g. window.alert("Invalid entry!").
  •  Events: An event is used to trap actions related to its owning object; typically, these actions are caused by the user.
    • For example, when the user clicks on a submit button, this is a click event which occurs at the submit object. By virtue of submitting a form, a submit event is also generated, following the click event.
    • Events can be intercepted and trigger specified program code to execute.

    • treats selection lists as an array of option values.

function checkSelection(sel)
{
   var message = "The " + sel.name + " list has " + 
                          sel.length + " options\n";
   message += "The list has option " + sel.selectedIndex + 
              " selected first\n";
   message += "The first selected option is " + 
               sel.options[sel.selectedIndex].text +"\n";
   message += "The first selected option's value is " + 
               sel.options[sel.selectedIndex].value  +"\n\n";
        
   // If there are multiple selection items allowed you need
   // to loop through the options array to get all of the 
   // selected values
   if(sel.multiple)
   {
      for(var i=0; i < sel.length; i++)
      {
        if(sel.options[i].selected)
        {
           message += "List options[" +i+ "].text is " + 
                       sel.options[i].text +"\n";
           message += "List options[" +i+ "].value is " + 
                       sel.options[i].value +"\n";
        }
      }
   }
   alert(message);
}

JavaScript: Events

  • The JavaScript Event Model exists so that scripts can respond to user actions and change the Web-page accordingly.  
  • Events occur when something happens within the browser window, either when the web page loads or when the user performs some action either with the mouse or the keyboard.

The Way Events Work

  • We capture events and trigger code using an event handler. 
    • An event handler is typically written as an attribute within the HTML tag for the element you are interested in watching. 
    • For instance, the following HTML tag defined an event handler for the Click event for a form field button named "clickme".
<input type="button" value="Click Me" 
       onClick="alert('message');">

  • Event handler attributes prepend the letters "on" to the event name; thus an event handler for a Click event is written as onClick. 

Events that Don't Call Functions

  • You needn't only call functions in the event handler -- any set of valid JavaScript statements will work. To alter the status message you can: 
    <input type="button" name="clickme"
    value="Click Here to See Results"
    onClick="window.status='Thanks';return true;">

  • We use single quotation marks (') within the event handler because the entire event handler code is contained within a set of double quotation marks(").
  • When specifying a new message for the status bar, returning a value of true keeps that message on the status bar after the event handler has completed.

  • The LOAD event fires whenever an element finishes loading successfully and is often used in the <BODY> tag to initiate scripts as soon as the page has been loaded into the window.

ONSUBMIT and ONRESET

  • Two more events that are useful when dealing with forms are onSubmit and onReset.  These events fire when the form is submitted or reset respectively.

  • onSubmit can also be used to redirect the browser to a confirmation page when the form is submitted successfully.

Variables: Duration and Scope (revisited)

  • Duration: the time during which a chunk of memory is allocated for that variable.
  • Scope: the part of the program where a variable can be evaluated or assigned.
  • Global variables are declared outside of a function, have scope from when they are declared until the document is unloaded and are available to an entire script
  • Local variables declared within in an individual function, have scope from when they are declared to when the function exits and are only available in the function in which they are defined

Function Overloading

  • Overloaded functions are typically different functions with the same name but different argument lists.
  • In many programming languages (Java, C++ ...) overloaded functions are used to perform similar operations on different data types or on different numbers of arguments.
  • This makes programs easier to read and understand.
    • eg. you can use a function Maximum to find the maximum value of either integers or floats or for 2, 3, 4 etc. numbers.
  • In JavaScript a single function can be written to work with different data types and different numbers of arguments!

Function Overloading Example

  • For example, a "confirmLink" function that confirms whether a person wants to visit a given link can be written to take no arguments (the link is hardcoded), one argument (a link to go to), or 2 arguments (the link to go to and a message to display).
  • In JavaScript if all 3 functions were written separately, included in the same file and given the same name - only the last one would be executed.
  • A JavaScript a single function can be designed to take 0, 1, 2 etc. arguments.

Arrays and Functions

Call-by-value

  • In JavaScript when an individual argument is passed to a function it is passed using "Call-by-value"
  • Call-by-value is a function call made by passing values of the expressions in the parentheses.
  • In call-by-value, the called function receives these values and copies them to the automatic variables in its argument list.
  • Advantage: The called function cannot change the values of the corresponding variables in the calling function.
  • Disadvantage: If a large object is being passed, making a copy can take extra memory space and execution time.

Call-by-reference

  • Arrays (and other Objects) in JavaScript use "Call-by-reference" when they are passed to a function.
  • Call-by-Reference does not make copies of original data
  • Call-by-Reference allow functions to access and manipulate common data resources quickly...
  • Call-by-value: transmits a copy of a value
  • Call-by-reference: transmits an address


Passing Arrays to Functions

  • To pass an array element as an argument to a function, specify the individual element (passed call-by-value).
    •    someFunction(response[4]);
  • To pass an entire array as an argument to a function, specify the name of the array without any brackets (passed call-by-reference)
    •    someFunction(response);
  • To receive an array, the function parameter list must specify that a argument will be received and then that argument must be used as an array within the function
    •    function someFunction(r)
       { document.write(r.length);}
